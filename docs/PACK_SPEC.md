# Language Pack Specification

This document defines the structure, schemas, and rules for LinguaForge Language Packs. A Language Pack is a collection of JSON files that provides all the content needed to learn a language -- vocabulary, grammar, characters, readings, roadmaps, and resources.

**No code required.** Pack contributors only write JSON.

## Pack Structure

```
src/packs/<language-id>/
  manifest.json              Required -- pack metadata and configuration
  vocabulary/
    <level>.json             Vocabulary entries per level (e.g. n5.json)
  grammar/
    <level>.json             Grammar points per level
  characters/
    <system>.json            Character entries per writing system (e.g. hiragana.json)
    <system>/
      <level>.json           Characters grouped by level (e.g. kanji/n5.json)
  readings/
    <level>.json             Reading passages per level
  roadmaps.json              Optional -- study roadmaps (intensive, standard, casual)
  resources.json             Optional -- recommended external resources
```

### Example: Japanese Pack

```
src/packs/japanese/
  manifest.json
  vocabulary/
    n5.json
    n4.json
    n3.json
  grammar/
    n5.json
    n4.json
    n3.json
  characters/
    hiragana.json
    katakana.json
    kanji/
      n5.json
      n4.json
      n3.json
  readings/
    n5.json
    n4.json
    n3.json
  roadmaps.json
  resources.json
```

## Writing Families

Every pack declares a `family` field that maps to a writing system adapter in the core engine. The adapter handles tokenization, furigana/pinyin, input conversion, and answer comparison.

| Family | Languages | Adapter Features |
|--------|-----------|-----------------|
| `cjk-japanese` | Japanese | Morphological tokenization (kuromoji), furigana, romaji-to-kana (wanakana), pitch accent |
| `cjk-chinese` | Mandarin, Cantonese | Tokenization (jieba), pinyin, tone marks |
| `hangul` | Korean | Hangul processing (es-hangul) |
| `latin` | English, Spanish, French, German, Italian, Portuguese, etc. | Space-based tokenization (built-in) |
| `cyrillic` | Russian, Ukrainian, etc. | Space-based tokenization |
| `arabic` | Arabic, Persian, etc. | Space-based tokenization + RTL support |
| `devanagari` | Hindi, Sanskrit, etc. | Space-based tokenization + transliteration |

## Manifest Schema

The `manifest.json` file is the heart of every pack. It declares the pack's identity, capabilities, and structure.

```jsonc
{
  // Required -- unique identifier, lowercase, no spaces
  "id": "japanese",

  // Required -- display name
  "name": "Japanese",

  // Required -- name in the target language
  "nativeName": "日本語",

  // Required -- emoji flag or icon identifier
  "icon": "🇯🇵",

  // Required -- semver version string
  "version": "1.0.0",

  // Required -- at least one author
  "authors": ["LinguaForge Team"],

  // Required -- license identifier
  "license": "MIT",

  // Required -- true if content was generated by AI
  "aiGenerated": false,

  // Required -- writing system family (see table above)
  "family": "cjk-japanese",

  // Required -- speech recognition and TTS configuration
  "speech": {
    "recognitionLang": "ja-JP",   // BCP-47 language tag
    "ttsLang": "ja-JP",           // BCP-47 language tag
    "defaultRate": 0.8            // 0.1 to 2.0
  },

  // Required -- text direction
  "direction": "ltr",             // "ltr" or "rtl"

  // Required -- writing systems used in this pack
  "writingSystems": [
    { "id": "hiragana", "name": "Hiragana", "hasDrawing": true },
    { "id": "katakana", "name": "Katakana", "hasDrawing": true },
    { "id": "kanji", "name": "Kanji", "hasDrawing": true, "hasStrokeOrder": true }
  ],

  // Required -- feature flags for the engine
  "features": {
    "pitchAccent": true,          // Show pitch accent data
    "toneMarks": false,           // Show tone marks (Chinese)
    "furigana": true,             // Show reading annotations
    "inputConversion": "wanakana", // Input conversion library or null
    "tokenization": true          // Enable morphological tokenization
  },

  // Required -- at least one level
  "levels": [
    { "id": "n5", "name": "JLPT N5", "order": 1, "description": "Beginner" },
    { "id": "n4", "name": "JLPT N4", "order": 2, "description": "Upper beginner" },
    { "id": "n3", "name": "JLPT N3", "order": 3, "description": "Intermediate" }
  ],

  // Required -- content categories present in this pack
  "categories": ["vocabulary", "grammar", "characters", "reading", "listening"]
}
```

### Writing System Object

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `id` | string | Yes | Unique identifier (e.g. `"hiragana"`, `"kanji"`) |
| `name` | string | Yes | Display name |
| `hasDrawing` | boolean | No | Whether character drawing exercises are available |
| `hasStrokeOrder` | boolean | No | Whether stroke order data is included |

### Feature Flags

| Field | Type | Description |
|-------|------|-------------|
| `pitchAccent` | boolean | Enable pitch accent display (Japanese) |
| `toneMarks` | boolean | Enable tone mark display (Chinese) |
| `furigana` | boolean | Enable reading annotations above text |
| `inputConversion` | string or null | Library for real-time input conversion (`"wanakana"` or `null`) |
| `tokenization` | boolean | Enable morphological tokenization |

## Vocabulary Schema

File: `vocabulary/<level>.json` -- an array of vocabulary entries.

```jsonc
[
  {
    // Required -- the word in the target language
    "word": "食べる",

    // Required -- phonetic reading
    "reading": "たべる",

    // Required -- at least one meaning (Spanish preferred, English accepted)
    "meanings": ["comer"],

    // Required -- part of speech tag
    "pos": "verb-ichidan",

    // Required -- must match a level id from the manifest
    "level": "n5",

    // Required -- tags for categorization (can be empty array)
    "tags": ["food", "daily-life"],

    // Required -- example sentences (can be empty array, 2+ recommended)
    "examples": [
      {
        "ja": "毎日ご飯を食べる。",
        "reading": "まいにちごはんをたべる。",
        "translation": "Como arroz todos los dias."
      }
    ],

    // Optional -- path to audio file
    "audio": "audio/n5/taberu.mp3",

    // Optional -- language-specific extra data
    "extra": {
      "pitchAccent": [0, 1, 0],
      "jlptFrequency": 1
    }
  }
]
```

### Part of Speech Tags

Common values: `noun`, `verb-ichidan`, `verb-godan`, `verb-irregular`, `adjective-i`, `adjective-na`, `adverb`, `particle`, `conjunction`, `interjection`, `prefix`, `suffix`, `counter`, `pronoun`, `expression`.

## Grammar Schema

File: `grammar/<level>.json` -- an array of grammar points.

```jsonc
[
  {
    // Required -- unique identifier
    "id": "n5-desu",

    // Required -- short title
    "title": "です",

    // Required -- grammar pattern
    "pattern": "Noun + です",

    // Required -- concise meaning
    "meaning": "Is / am / are (polite)",

    // Required -- must match a level id
    "level": "n5",

    // Required -- detailed explanation
    "explanation": "です is the polite copula used to equate or describe nouns...",

    // Required -- at least one example
    "examples": [
      {
        "sentence": "学生です。",
        "reading": "がくせいです。",
        "translation": "Soy estudiante.",
        "breakdown": ["学生 = estudiante", "です = soy/es (polite)"]
      }
    ],

    // Optional
    "notes": "In casual speech, です can be dropped or replaced with だ.",
    "related": ["n5-ja-nai-desu", "n5-deshita"],
    "tags": ["copula", "polite"]
  }
]
```

## Character Schema

File: `characters/<system>.json` or `characters/<system>/<level>.json` -- an array of character entries.

```jsonc
[
  {
    // Required -- the character itself
    "character": "食",

    // Required -- at least one reading
    "readings": [
      { "type": "on", "value": "ショク" },
      { "type": "kun", "value": "た.べる" }
    ],

    // Required -- at least one meaning
    "meanings": ["eat", "food"],

    // Required -- number of strokes
    "strokeCount": 9,

    // Optional -- SVG path data or reference
    "strokeOrder": "M...",

    // Optional -- radical information
    "radical": "食",
    "radicalMeaning": "eat, food",

    // Required -- must match a level id
    "level": "n5",

    // Optional
    "tags": ["food"],

    // Optional -- word examples using this character
    "examples": [
      { "word": "食べる", "reading": "たべる", "meaning": "to eat" },
      { "word": "食事", "reading": "しょくじ", "meaning": "meal" }
    ],

    // Optional -- language-specific extra data
    "extra": {
      "jlptFrequency": 32
    }
  }
]
```

### Reading Types

| Type | Used For |
|------|----------|
| `on` | On'yomi (Chinese-origin reading) for kanji |
| `kun` | Kun'yomi (native Japanese reading) for kanji |
| `nanori` | Name readings for kanji |
| `romaji` | Romanization for kana characters |

## Reading Schema

File: `readings/<level>.json` -- an array of reading passages.

```jsonc
[
  {
    // Required -- unique identifier
    "id": "n5-self-introduction",

    // Required -- passage title
    "title": "Self Introduction",

    // Required -- must match a level id
    "level": "n5",

    // Required -- the passage text
    "text": "はじめまして。田中太郎です。東京に住んでいます。",

    // Optional -- full phonetic reading
    "reading": "はじめまして。たなかたろうです。とうきょうにすんでいます。",

    // Optional -- full translation
    "translation": "Mucho gusto. Soy Taro Tanaka. Vivo en Tokio.",

    // Optional -- vocabulary words used in this passage
    "vocabulary": ["はじめまして", "住む", "東京"],

    // Optional -- comprehension questions
    "questions": [
      {
        "question": "Where does Taro live?",
        "options": ["Osaka", "Tokyo", "Kyoto", "Nagoya"],
        "correctIndex": 1,
        "explanation": "The text says 東京に住んでいます (lives in Tokyo)."
      }
    ]
  }
]
```

## Roadmap Schema

File: `roadmaps.json` -- an array of study roadmaps.

```jsonc
[
  {
    "id": "standard-jlpt",
    "name": "Standard JLPT Path",
    "type": "standard",           // "intensive" | "standard" | "casual"
    "description": "A balanced study plan covering all JLPT levels.",
    "phases": [
      {
        "name": "Foundation",
        "duration": "3 months",
        "level": "n5",
        "content": "Hiragana, Katakana, basic vocabulary and grammar",
        "dailyGoal": "30 minutes, 20 new cards",
        "milestones": [
          "Master all hiragana and katakana",
          "Learn 100 basic vocabulary words",
          "Understand basic sentence patterns"
        ]
      }
    ]
  }
]
```

## Resource Schema

File: `resources.json` -- an array of recommended external resources.

```jsonc
[
  {
    "name": "Genki I",
    "author": "Eri Banno et al.",
    "type": "book",               // "book" | "video" | "audio" | "app" | "website" | "method"
    "level": "n5",
    "areas": ["grammar", "vocabulary", "reading"],
    "instructionLanguage": "en",
    "url": "https://genki3.japantimes.co.jp/",
    "price": "paid",              // "free" | "paid" | "freemium"
    "priceApprox": "~$50 USD",
    "communityRating": "4.8/5",
    "description": "The most popular Japanese textbook series for beginners.",
    "pros": ["Well-structured", "Comprehensive exercises", "Audio included"],
    "cons": ["English-only instruction", "Requires self-discipline"],
    "recommendedOrder": 1,
    "complementaryResources": ["Genki Workbook I"]
  }
]
```

## Creating a New Pack

1. Create a directory: `src/packs/<language-id>/`
2. Write `manifest.json` following the schema above
3. Add content files in the appropriate subdirectories
4. Validate your pack:
   ```bash
   npm run validate-pack -- --pack src/packs/<language-id>
   ```
5. Submit a pull request

### Naming Conventions

- **Pack directory:** Lowercase English name of the language (e.g. `japanese`, `mandarin`, `korean`, `spanish`)
- **Level files:** Use the level `id` from your manifest as the filename (e.g. `n5.json`, `a1.json`, `beginner.json`)
- **Character files:** Use the writing system `id` from your manifest (e.g. `hiragana.json`, `kanji/n5.json`)

### Tips

- Start with a single level. You can always add more later.
- Include at least 2 example sentences per vocabulary entry and grammar point.
- Provide both Spanish and English meanings when possible.
- Set `"aiGenerated": true` in the manifest if you used AI to generate content.
- Run validation before submitting. It catches most issues.

## Validation

All pack files are validated against Zod schemas defined in `src/validation/pack-schema.ts`.

```bash
# Validate a specific pack
npm run validate-pack -- --pack src/packs/japanese

# The validator checks:
# - manifest.json structure and required fields
# - All vocabulary, grammar, character, and reading files
# - roadmaps.json and resources.json (if present)
# - Level references match manifest definitions
# - Version string format (semver)
# - Array minimums (at least 1 author, 1 level, 1 writing system, etc.)
```

## AI-Generated Content Guidelines

AI tools can accelerate pack creation, but accuracy is critical for language learning.

1. **Always set `"aiGenerated": true`** in the manifest if any content was AI-generated
2. **Cross-reference readings and meanings** against authoritative sources (dictionaries, official word lists)
3. **Verify example sentences** are natural and grammatically correct
4. **Check stroke counts** against a kanji dictionary
5. **Review pitch accent data** if included -- AI frequently gets this wrong
6. **Have a native speaker or advanced learner review** the content before submitting

See [PACK_CREATOR_PROMPT.md](PACK_CREATOR_PROMPT.md) for a ready-to-use prompt template for generating pack data with LLMs.
